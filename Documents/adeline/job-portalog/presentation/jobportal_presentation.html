JobPortal Rwanda

A role-based React web application connecting applicants and employers through a modern, data-driven recruitment workflow.

Project Goal

Build a complete frontend hiring platform where:

  • Applicants can discover jobs and apply with CV upload.
  • Employers can post jobs, manage applications, and review analytics.
  • Both roles get a clear, responsive, and secure navigation experience.

Business Value

  • Reduces friction between job posting and candidate submission.
  • Provides immediate visibility into recruitment activity.
  • Improves candidate quality with built-in validation and structured forms.

System Design

Architecture & Key Features

The app is organized around protected routes, reusable components, and a shared global context for state, theme, and workflow actions.

Role-Based Routing

  • `/employer/*` protected for employer role only.
  • `/applicant/*` protected for applicant role only.
  • Automatic redirects on unauthorized access.

Global State (AppContext)

  • Auth state, jobs, applicants, and theme mode.
  • CRUD operations for jobs and applications.
  • Data persisted in localStorage with TTL strategy.

Applicant Experience

  • Job search with filters (type, industry, keyword).
  • Job detail modal and one-click application flow.
  • CV upload with PDF + size validation (max 2MB).

Employer Experience

  • Manage jobs with DataGrid actions (view/edit/delete).
  • Add/Edit posting with salary & required field validation.
  • Analytics dashboard with Bar, Pie, Area, and Line charts.

Quality Highlights

  • Responsive layout behavior across desktop/tablet/mobile breakpoints.
  • Dark/Light mode support with centralized MUI theming.
  • Reusable dialog components for detail views and confirmations.
  • Clear component separation across pages, charts, dialogs, and layout.
